# Contributing to NovaLMS

Thank you for considering contributing! Please follow these guidelines:

- Fork the repository and create your branch from `main`.
- If you've added code, ensure it is well-tested.
- If you've changed APIs, update the documentation.
- Open a pull request with a clear description of your changes.

## Code Style

- Use consistent formatting and naming conventions.
- Write clear, concise commit messages.

## Issues

- For bug reports, include steps to reproduce and screenshots if possible.
- For feature requests, describe the use case and proposed solution.
